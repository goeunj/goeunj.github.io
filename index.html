Canvas Calendar UI Design: User Interface Design
May 2021 - Aug 2021

  • Discussed good and bad design elements the Canvas calendar had to create 3 functional and 3 non-functional changes to enhance the overall design
  • Created 80% of the medium fidelity prototype in Figma by designing the look of our design and including the interactive portion

This was a project from a 'User Interface Design' course. In a group of 5, we took a look at the SFU Canvas calendar to create changes that would increase the inclusivity and efficiency of the calendar layout.
To do so, we had 3 iterations.

  1. Come up with the Good vs the Bad design
  2. Choose 3 functional and non-functional requirements to add to the interface
    • Create a low fidelity prototype in Balsamiq
    • Create a medium fidelity prototype in Figma
  3. Gather user experience reviews

Iteration 1

Good Design

  • Overall design is very clear and simple
  • A confirmation pop-up appears when deleting an event for error precention

Bad Design

  • Lack of indication on how to hide specific courses
  • User is unable to add a new calendar
  • Lack of flexibility in choosing specific months

Iteration 2

3 Functional Requirements

  • Make choosing months more efficient
  • Showing and hiding a calendar should be more apparent to the user
    • User should have an easier way to choose calendar colour
  • User should be able to add their own calendar and add events

3 Non-Functional Requirements

  • User should not be able to add events to calendars they do not own
    • i.e., only instructors have access to adding, deleting or changing events on course calendars
  • User should be able to use keyboard shortcuts to navigate the interface
  • User should have an option to read aloud calendar elements

Low Fidelity Prototype

  • Clicking on the month/year button in the top-left corner of the screen allows the user to quickly navigate to a different month for a given year
  • With the addition of a swuare icon next to a course (or calendar name), the user can easily hide or show the calendar events
  • The square '+' icon next to the 'Calendars' heading in the side panel allows the user to create a new calendar group, with a selected colour

Medium Fidelity Prototype

Original View



The following keyboard shortcuts are supported:
  • Press 'j' or 'n' to go to the next month
  • Press '2' or 'w' to go to week view
  • Press '3' or 'm' to go to month view
  • Press 'c' to create a new calendar
  • Press 'e' to see event details

This is an example of what pressing '2' or 'w' would look like. Clicking '3' or 'm' would return to the orginal view shown above.



To add a new calendar, user has to click the square '+' icon next to the 'Calendars' heading in the side panel.



This shows an error message because a calendar with the same name already exists.



Changing the name gets rid of the error message.



Clicking submit from the previous image adds the new calendar to the list of calendars in the side panel.

Iteration 3

We held a cognitive walkthrough of our medium fidelity prototype and gave our user the following scenario:

"You are a 1st year student attending SFU. You have chosen your classes and have signed up for a club. Looking at your calendar, you notive that the club is not showing up because it is not an official course at SFU. So, you decide to add a new calendar to represent the club."

We then gave the user 5 actions to complete without telling them the exact buttons to click.

  1. Get "Add New Calendar" pop-up to display
  2. Cancel adding a new calendar
  3. Submit new calendar
  4. Hide/show one of current calendars
  5. Select January as the current month
Overall, the user had no difficulties carrying out each task. Our prototype had very apparent and well placed functionalities. We also used icons that people are alredy familiar with. However, the user did take some time figuring out that the month/year had a drop down menu. If we had another iteration, we would take these results and make the appropriate changes.

Mar 2021 - Apr 2021

The storage server was taken down by the instructor and the web application no longer works. However, below are some images that showcase the overall look and functionalities of the web application.
